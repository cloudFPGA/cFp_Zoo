12 // This header includes a module that allows pipelined accumulation of data
13 // types with a non-zero latency on the operation, such as addition of floating
14 // point numbers, using a minimal amount of arithmetic units.
15 // The functionality is split into three functions:
16 //
17 // 1) Iterate, which accumulates `latency` different inputs, outputting
18 // partial results.
19 // 2) Feedback, which loops partial results back to Iterate to continue
20 // accumulation until done, then emits the result.
21 // 3) Reduce, which finally collapses the `latency` inputs into a single
22 // output.
23 //
24 // If latency is not set high enough, the design will hang, as no new values
25 // will be available from Feedback when Iterate requires them, and the pipeline
26 // will stall forever. The latency should be at least the sum of the latencies
27 // of the Iterate and Feedback modules, plus a few cycles for the overhead of
28 // the FIFOs between them.
29 // For example, for single precision floating point addition at 300 MHz, the
30 // latency of Iterate is 10 and the latency of bounce is 2. A latency of 14 is
31 // enough to successfully run accumulation.
32 //
33 // Due to a bug where Vivado HLS (as of 2017.1) considers all streams static,
34 // the three functions cannot be factored into a generic function here, as
35 // calling it multiple times would result in dataflow errors on the internal
36 // streams. Instead call them as follows:
37 //
38 // hlslib::Stream<T> toFeedback("fromFeedback");
39 // hlslib::Stream<T> fromFeedback("fromFeedback");
40 // #pragma HLS STREAM variable=fromFeedback depth=latency
41 // Stream<T> toReduce("toReduce");
42 // HLSLIB_DATAFLOW_FUNCTION(hlslib::AccumulateIterate, in, fromFeedback,
43 // toFeedback);
44 // HLSLIB_DATAFLOW_FUNCTION(hlslib::AccumulateFeedback, toFeedback,
45 // fromFeedback, toReduce);
46 // HLSLIB_DATAFLOW_FUNCTION(hlslib::AccumulateReduce(toReduce, out);
47 //
48 // For convenience, a function for reducing with a single cycle latency
49 // operation is also included as AccumulateSimple.
